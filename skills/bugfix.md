# Bugfix Skill

## Description
버그를 분석하고 수정하는 스킬입니다.

## When to use
- 에러 수정 요청
- 동작 오류 해결
- 예외 처리 개선

## Instructions
1. 에러 분석
   - 에러 메시지/증상 파악
   - 스택 트레이스 확인
   - 재현 조건 이해

2. 원인 파악
   - 관련 코드 추적
   - 데이터 흐름 분석
   - 엣지 케이스 확인

3. 수정
   - 최소 범위로 수정
   - 루트 원인 해결 (임시 땜빵 X)
   - 사이드 이펙트 확인

4. 검증
   - 버그 재현 불가 확인
   - 관련 기능 동작 확인

## TDD Mode (기본 적용)
버그 수정은 TDD 방식을 따릅니다:
1. **재현 테스트 작성** → 버그 재현 확인 (Red)
2. **버그 수정** → 테스트 통과 (Green)
3. **리팩토링** → 테스트 유지 (Refactor)

재현 테스트가 먼저 있어야 수정 확인이 가능합니다.
상세 가이드: `tdd.md` 스킬 참조

## Principles
- 기존 동작 유지
- 최소 변경
- 루트 원인 해결

## Output
- 수정된 코드
- 버그 원인 설명 (주석 또는 커밋 메시지용)
