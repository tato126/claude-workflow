# Design Skill

## Description
시스템/기능 설계 문서를 작성하는 스킬입니다.

## When to use
- 새 기능 설계 요청
- 아키텍처 설계
- API 스펙 정의
- 데이터 모델 설계
- 시스템 흐름 설계

## Instructions
1. 요구사항 분석
   - 핵심 기능 파악
   - 비기능 요구사항 (성능, 확장성)
   - 제약 조건 확인

2. 현황 분석
   - 기존 시스템 구조 파악
   - 연관 컴포넌트 식별
   - 영향 범위 분석

3. 설계 작성
   - 개요 및 목적
   - 시스템 구조도
   - 데이터 흐름
   - 인터페이스 정의
   - 에러 처리 전략

4. 구현 가이드
   - 단계별 구현 순서
   - 의존성 정의
   - 테스트 전략

## Output Format
```markdown
# [기능명] 설계서

## 개요
- 목적:
- 범위:

## 요구사항
### 기능 요구사항
- [ ] 요구사항 1
- [ ] 요구사항 2

### 비기능 요구사항
- 성능:
- 보안:

## 시스템 구조
[다이어그램 또는 설명]

## 데이터 모델
[ERD 또는 스키마]

## API/인터페이스
[엔드포인트 또는 메서드 정의]

## 구현 계획
1. Phase 1: ...
2. Phase 2: ...

## 리스크
- 리스크 1 → 대응 방안
```

## Conventions
- 다이어그램은 Mermaid 또는 ASCII로 표현
- 기술 용어 일관되게 사용
- 버전/날짜 명시
