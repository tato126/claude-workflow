# Code Review Skill

## Description
작성된 코드를 리뷰하고 개선점을 제안하는 스킬입니다.

## When to use
- REVIEW 상태 태스크
- 수동 리뷰 요청
- PR 전 자체 검토

## Checklist
- [ ] 타입 안전성
- [ ] 에러 핸들링
- [ ] 성능 이슈
- [ ] 보안 취약점
- [ ] 코드 중복
- [ ] 네이밍 명확성
- [ ] 테스트 충분성
- [ ] 문서화 필요성

## Review Areas

### 1. 코드 품질
- 가독성
- 유지보수성
- 일관성

### 2. 기능성
- 요구사항 충족
- 엣지 케이스 처리
- 에러 핸들링

### 3. 성능
- 불필요한 연산
- 메모리 사용
- 최적화 가능 부분

### 4. 보안
- 입력 검증
- 인증/인가
- 민감 정보 노출

## Output Format
```markdown
## 리뷰 결과

### 🔴 Critical (반드시 수정)
- 이슈 설명
- 수정 제안

### 🟡 Warning (권장 수정)
- 이슈 설명
- 수정 제안

### 🟢 Suggestion (개선 제안)
- 제안 내용
```
