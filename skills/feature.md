# Feature Development Skill

## Description
새로운 기능을 개발하는 스킬입니다.

## When to use
- 새 기능 추가 요청
- API 엔드포인트 구현
- UI 컴포넌트 개발
- 새로운 모듈/서비스 추가

## Instructions
1. 요구사항 분석
   - 프롬프트에서 핵심 요구사항 파악
   - 필요한 입출력 정의

2. 기존 코드 패턴 확인
   - 프로젝트의 기존 구조 파악
   - 유사한 기능의 구현 방식 참고
   - 네이밍 컨벤션 확인

3. 구현
   - 단일 책임 원칙 준수
   - 작은 함수로 분리
   - 명확한 변수/함수명 사용

4. 타입 정의
   - TypeScript 사용 시 strict 타입 적용
   - 인터페이스/타입 명확히 정의

5. 에러 핸들링
   - 예외 상황 처리
   - 사용자 친화적 에러 메시지

## TDD Mode (기본 적용)
모든 기능 개발은 TDD 방식을 따릅니다:
1. 테스트 먼저 작성 → 실패 확인 (Red)
2. 최소 구현 → 테스트 통과 (Green)
3. 리팩토링 → 테스트 유지 (Refactor)

상세 가이드: `tdd.md` 스킬 참조

## 엣지 케이스 테스트 (필수)
모든 기능 개발 시 엣지 케이스를 식별하고 테스트합니다:
1. 정상 케이스 테스트
2. 경계값 테스트 (최소/최대/빈값)
3. 예외 상황 테스트 (null, 잘못된 입력 등)
4. 동시성 이슈 가능성 검토
5. 의존성 실패 시나리오 (외부 API, DB 등)

## Conventions
- 프로젝트의 기존 패턴 따르기
- ESLint/Prettier 규칙 준수
- 주석은 "왜"에 집중 (코드가 "무엇"을 설명)

## 완료 조건 (필수)
1. 모든 테스트 통과 (유닛 + 엣지 케이스)
2. 빌드 성공
3. Confluence에 작업 내용 상세 문서화
   - 구현 개요
   - 변경된 파일 목록
   - API 변경 사항 (있는 경우)
   - 테스트 케이스 목록
   - 엣지 케이스 처리 방법

## Output
- 구현된 코드
- 필요시 타입 파일
- 수정된 기존 파일
- 테스트 코드 (엣지 케이스 포함)
- Confluence 문서
